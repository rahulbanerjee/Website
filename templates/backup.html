Class-based generic views

Django ships with generic views to do the following:

  • Perform common “simple” tasks: redirect to a different page and render a given template.
  • Display list and detail pages for a single object. If we were creating an application to manage conferences then a TalkListView and a RegisteredUserListView would be examples of list views.

Taken together, these views provide easy interfaces.

Simple usage

You can override attributes (such as the template_name) or methods (such as get_context_data) in your subclass to provide new values or methods.
